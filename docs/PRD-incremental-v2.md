> **版本**：v2-incremental
> **日期**：2026-06-26

> **文档性质**：增量文档，仅描述本次 4 项变更，不重复已有 PRD 内容

## 1. 变更目标

本次改造解决 v1.0 的三个核心局限：**型号硬编码、单卫星假设、导航作用域混乱**，并补齐**批量数据录入手段**。具体达成：

| # | 目标 | 对应变更 | 衡量标准 |
|---|------|----------|----------|
| **CG1** | 系统去型号化，成为可承载任意卫星制造项目的通用平台 | 变更一 | 全代码无 "灵犀10B" 字面量；项目名/卫星型号均由运行时配置 |
| **CG2** | 以「项目」为顶层组织单元，单项目支持 1~N 颗卫星并行管理 | 变更二 | `Project.satellites: Satellite[]`；可新建项目、项目内切换卫星 |
| **CG3** | 导航按「全局 → 项目 → 卫星 → 子功能」分层，齐套/阶段/AIT 数据按项目+卫星作用域隔离 | 变更三 | 切换卫星后，三项数据互不串扰；未选项目时不可进入项目内功能 |
| **CG4** | 提供 Excel 模板化录入，单星单 sheet、多星多 sheet，导入即建树 | 变更四 | 下载模板 → 填写 → 上传 → 自动构建层级树并更新 BOM |

## 2. 变更需求清单

### 2.1 变更一：通用化（去型号硬编码）

| ID | 优先级 | 需求 | 描述 | 验收标准 |
|----|--------|------|------|----------|
| **C1-1** | P0 | 去除型号字面量 | 全局搜索清除 "灵犀10B" 等硬编码字符串（含顶栏文案、状态栏、Hero、Mock 数据默认值） | `grep -ri "灵犀10B" src/` 仅命中示例数据文件，非 UI/逻辑代码 |
| **C1-2** | P0 | 系统标题通用化 | 顶栏/页签/文档标题统一为「卫星制造项目管理系统」，不绑定型号 | 所有界面无固定型号文案；标题随当前项目动态显示 |
| **C1-3** | P0 | 项目/卫星名称运行时配置 | `Project.name`、`Satellite.name`、`Project.satelliteModel` 由用户新建项目时填写或由 Excel 导入生成，不写死 | 新建项目流程需输入项目名；无任何代码常量提供默认型号 |
| **C1-4** | P1 | Mock 数据参数化 | 现有 `bom_tree.json`（灵犀10B）降级为「示例数据集」，作为新建项目时的可选种子数据，而非唯一数据源 | 系统可脱离灵犀10B 数据独立运行；示例数据可一键导入为某项目 |

### 2.2 变更二：项目 + 多星数据模型

| ID | 优先级 | 需求 | 描述 | 验收标准 |
|----|--------|------|------|----------|
| **C2-1** | P0 | 数据模型复数化 | `Project.satellites: Satellite[]`（由单数 `satellite` 改为复数数组）；单星项目数组长度为 1 | TS 类型更新；旧字段兼容迁移或一次性切换 |
| **C2-2** | P0 | 项目列表页 | 新增「项目管理」页：展示所有项目卡片/表格（项目名、卫星数、物料总数、齐套率、更新时间），支持新建项目 | 空态有引导；新建弹窗含项目名/型号/卫星数；列表数据来自 store |
| **C2-3** | P0 | 卫星切换器 | 进入项目后，顶部/侧边提供卫星切换控件（Dropdown 或 Tab），切换后整页数据作用域刷新 | 切换卫星 ≤200ms 内刷新；当前选中卫星高亮 |
| **C2-4** | P0 | 作用域隔离 | 每颗卫星独立维护：层级树 / 齐套看板 / 阶段管理 / AIT 编排 | 切换卫星后，四项数据完全独立，无残留 |
| **C2-5** | P1 | 项目级聚合指标 | 项目总览页展示项目下所有卫星的聚合指标（总物料、加权齐套率、卫星进度对比） | 多星项目总览页可对比各卫星齐套率 |

### 2.3 变更三：导航重构（两层导航）

#### 2.3.1 新导航结构

```
顶层（全局，无需选项目）
├── 总览          ← 全局仪表盘，聚合所有项目
└── 项目管理      ← 项目列表 + 新建

项目内（选中项目后激活，按当前卫星作用域）
└── 层级数据（主入口，默认页）
    ├── BOM 齐套（子页）
    ├── 阶段管理（子页）
    └── AIT 编排（子页）
```

| ID | 优先级 | 需求 | 描述 | 验收标准 |
|----|--------|------|------|----------|
| **C3-1** | P0 | 顶层导航两项 | Sidebar 顶层仅保留「总览」「项目管理」；原 5 项平铺取消 | 未选项目时，项目内导航不可见/禁用 |
| **C3-2** | P0 | 层级数据为父级 | 「层级数据」作为项目内主入口，BOM齐套/阶段管理/AIT编排 收为其子页 | 子页以缩进或二级菜单呈现；URL/状态可直达子页 |
| **C3-3** | P0 | 项目+卫星作用域绑定 | 齐套/阶段/AIT 的 store 取数均带 `projectId` + `satelliteId` 参数 | 切换项目或卫星，数据自动重载对应作用域 |
| **C3-4** | P0 | 导航状态机 | `PageType` 扩展为两级：`{ scope: 'global'\|'project', page: ... }`；未选项目时项目内页不可达 | 直接访问项目内页但无选中项目时，重定向到项目管理 |
| **C3-5** | P1 | 面包屑 | 顶栏下方显示「项目管理 > [项目名] > [卫星名] > 当前页」 | 任意层级可点击回溯 |

#### 2.3.2 与 v1.0 导航对照

| v1.0（5 项平铺，全局） | v2（两层，作用域化） |
|------------------------|---------------------|
| 总览 | 总览（全局） |
| 层级数据 | 层级数据（项目内·主入口） |
| BOM 齐套 | └ BOM 齐套（项目内·子页） |
| 阶段管理 | └ 阶段管理（项目内·子页） |
| AIT 编排 | └ AIT 编排（项目内·子页） |
| — | 项目管理（全局，新增） |

### 2.4 变更四：Excel 输入模式

| ID | 优先级 | 需求 | 描述 | 验收标准 |
|----|--------|------|------|----------|
| **C4-1** | P0 | 通用模板下载 | 提供 `.xlsx` 模板下载入口（项目设置 / 层级数据页工具栏），含使用说明 sheet | 任意项目可下载空白模板 |
| **C4-2** | P0 | 模板 sheet 规则 | 单星 = 1 个 sheet；多星 = 多个 sheet；**sheet 名 = 卫星名** | 模板含明确 sheet 命名规则注释 |
| **C4-3** | P0 | 列结构定义 | 每个 sheet 列对应 BOM 字段 + 扩展字段，固定列序（见下方列定义表） | 导入时按列名映射，缺失列给警告 |
| **C4-4** | P0 | Excel 上传导入 | 上传 `.xlsx`，解析所有 sheet，每个 sheet → 一颗卫星的 BOM，更新到当前项目 | 导入后该项目 satellites 数量 = sheet 数 |
| **C4-5** | P0 | 自动构建层级树 | 导入后根据「层级(level)/料号/父子关系」字段自动重建五层级树 | 导入完成即可在层级数据页看到完整树 |
| **C4-6** | P1 | 导入预览与校验 | 上传后先展示预览（卫星数、每星物料数、异常行），确认后落库 | 校验失败行标红，可忽略或修正后重传 |
| **C4-7** | P1 | 导出当前项目 | 可将当前项目所有卫星导出为多 sheet Excel（与模板格式一致） | 导出文件可重新导入还原数据 |
| **C4-8** | P2 | 模板版本管理 | 模板可迭代，导入时识别模板版本号做兼容 | 新旧模板均可导入 |

#### 2.4.1 Excel 模板列结构定义

| 列序 | 列名（中文） | 字段 | 必填 | 说明 |
|------|-------------|------|------|------|
| A | 层级 | `level` | ✅ | 1=整星, 2=分系统, 3=单机/零部件 |
| B | 料号 | `part_no` | ✅ | 唯一标识 |
| C | 品名 | `name` | ✅ | — |
| D | 规格 | `spec` | — | — |
| E | 封装形式 | `form` | — | — |
| F | 厂家 | `manufacturer` | — | — |
| G | 质量等级 | `quality_level` | — | — |
| H | 用量 | `quantity` | — | 数值 |
| I | 位号 | `location` | — | — |
| J | 单位 | `unit` | — | — |
| K | 类型 | `type` | — | equipment/part，可由料号前缀推断 |
| L | 电性件齐套 | `electrical` | — | 是/否 |
| M | 鉴定件齐套 | `qualification` | — | 是/否 |
| N | 正样件齐套 | `flight` | — | 是/否 |
| O | 在整星中 | `in_satellite` | — | flight/qualification，逗号分隔 |
| P | 投产状态 | `production_status` | — | 未投产/生产中/已完成 |
| Q | 交付日期 | `delivery_date` | — | YYYY-MM-DD |

> **多星说明**：每个 sheet 独立描述一颗卫星的完整 BOM；sheet 名即卫星名，导入时自动创建 `Satellite` 实体并归入当前项目。

## 4. 待确认问题

| # | 问题 | 影响 | 建议 |
|---|------|------|------|
| **IQ1** | **项目与卫星的命名规则**：项目名是否需要唯一？同一型号卫星能否属于不同项目（如不同批次）？ | 数据模型、唯一约束 | 建议项目名唯一；型号可重复，卫星按项目隔离 |
| **IQ2** | **多卫星间数据关系**：同一项目下多颗卫星是独立 BOM，还是存在「基准星 + 派生星」复制关系？ | 导入逻辑、变更传播 | 建议默认独立；是否需要「复制卫星」功能待定 |
| **IQ3** | **Excel 层级表示方式**：当前模板用 `level` 数字表示层级，父子关系靠顺序推断。是否需要显式「父料号」列以支持乱序填写？ | 模板设计、解析复杂度 | 建议增加「父料号」可选列，提升容错 |
| **IQ4** | **导入冲突策略**：Excel 导入时若项目已有数据，是「覆盖」「合并」还是「新建卫星追加」？ | 导入逻辑、数据安全 | 建议默认「新建卫星追加」，覆盖需二次确认 |
| **IQ5** | **全局总览聚合粒度**：全局总览页是按项目聚合，还是直接列出所有卫星？ | 总览页设计 | 建议按项目聚合为卡片，点击展开看卫星 |
| **IQ6** | **卫星切换器位置**：放顶栏还是侧边栏？多星较多时（如 10+）是否影响布局？ | UI 布局 | 建议侧边栏 Dropdown；超 5 颗时改为搜索式选择器 |
| **IQ7** | **示例数据归属**：灵犀10B 示例数据是作为内置「示例项目」预置，还是仅在新建时可选种子？ | 初始化流程 | 建议首次进入系统预置 1 个「示例项目」便于演示 |
| **IQ8** | **Excel 库选型**：前端纯解析（SheetJS/xlsx）还是需后端？Demo 阶段是否接受纯前端解析？ | 技术实现 | 建议 Demo 用 SheetJS 前端解析，无需后端 |

## 5. 不变更范围（保持 v1.0 现状）

以下功能/规范在本次变更中**保持不变**，不在改造范围内：

| 类别 | 保持不变的内容 |
|------|----------------|
| **技术栈** | Vite + React + TypeScript + MUI + Tailwind + Zustand + @dnd-kit + Recharts |
| **视觉风格** | 苹果官网设计语言（大留白、圆角卡片、毛玻璃、微妙阴影） |
| **日夜切换** | 日/夜模式一键切换，无闪烁过渡，色板不变 |
| **五层级树** | 批次→整星→分系统→单机→元器件 的层级结构模型不变（仅数据源多星化） |
| **单机三状态** | 电性件/鉴定件/正样件 的三状态管理逻辑不变 |
| **齐套看板** | 分系统维度齐套率 + 色块标识 + 三状态筛选 + 下钻逻辑不变 |
| **四阶段框架** | 设计/投产/联试/AIT 四阶段及各阶段视图内容不变 |
| **AIT 拖拽编排** | 6 类预置工作 + @dnd-kit 拖拽排序 + 跨列移动 + 新增工作不变 |
| **多端适配** | Mac/Windows/移动端 Chrome 响应式适配标准不变 |
| **Service 抽象** | DataService 接口层保留，便于后续接 MES/ERP（仅扩展为多项目/多星取数） |
| **P2 远期项** | 自动取数、钉钉协作、元器件层级、权限、甘特图等仍为后续迭代 |
